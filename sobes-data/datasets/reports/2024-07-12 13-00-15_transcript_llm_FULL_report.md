## Блок 3: Глубина копания по темам

### Темы с максимальным копанием:
- React (хуки, пробрасывание пропсов, менеджеры состояния, архитектура уведомлений) — 6 вопросов с уточнениями и обсуждением
- JavaScript (event loop, циклы и рекурсия, сортировка, алгоритмы) — 7 вопросов с пошаговым разбором и уточнениями
- Архитектура (FSD, микрофронты, NPM-пакеты) — 4 вопроса с обсуждением преимуществ и недостатков
- Авторизация и безопасность (токены, хранение) — 3 вопроса с деталями
- Тестирование (виды, польза) — 2 вопроса
- Оптимизация фронтенда (lazy loading, svg, debounce) — 3 вопроса

### Паттерны углубления:
- По React: от вопроса про используемые хуки к разбору debounce → обсуждение размонтирования компонента → проблема проброса пропсов → менеджеры состояний → реализация уведомлений в рамках FSD → организация shared кода
- По JavaScript: общий вопрос о работе event loop → уточнение микрозадач и макрозадач → вопрос о бесконечном цикле vs рекурсии с объяснением переполнения стека → практические вопросы по сортировке и оценке сложности алгоритма → обсуждение оптимального решения задачи слияния массивов
- По архитектуре: от вопроса о микрофронтах к сравнению с NPM-пакетами → обсуждение ручного обновления зависимостей → связь с FSD → разделение кода и проблемы с обновлениями
- По безопасности: от опыта авторизации к типам токенов и хранению → обсуждение недостатков localStorage → соображения по безопасности

### Как интервьюер интересуется опытом кандидата:
- Начинает с общего вопроса про опыт и роль кандидата
- Уточняет детали по проектам (админки, NPM-пакеты, FSD)
- Интересуется конкретными технологиями и подходами, которые кандидат применял
- Задает вопросы по реальным задачам (оптимизация, авторизация, архитектура)
- В конце проверяет алгоритмические навыки через задачу на слияние массивов
- Спрашивает о тестировании и работе в команде
- Поддерживает диалог, уточняя детали и предлагая рассуждать над алгоритмами
